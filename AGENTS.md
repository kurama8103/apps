# AGENTS.md

このドキュメントは、AI開発エージェントがこのプロジェクトで作業する際のガイドラインを提供します。

## プロジェクト概要

このプロジェクトは、仮想通貨取引所間の裁定取引機会を特定し、関連するデータ分析を行うためのアプリケーションです。コマンドラインツールとしてのコア機能と、Streamlitを利用したWeb UIの両方を提供します。

プロジェクトの目的、セットアップ方法、使い方についての詳細は、ルートディレクトリにある `README.md` を参照してください。

## プロジェクト構成

このプロジェクトは、Streamlitのマルチページアプリケーションとして構成されています。

-   `/`: プロジェクトのルートディレクトリ。Streamlitアプリケーションのエントリーポイントである`index.py`、設定ファイル`config.ini`、依存関係を定義する`requirements.txt`が配置されています。
-   `/pages`: Streamlitの各サブページ（アプリケーション）が格納されています。仮想通貨関連の機能もこのディレクトリ内の一つのページとして実装されています。
-   `/core`: 裁定取引の計算やデータ取得など、アプリケーション全体で共有される中核的なビジネスロジックが含まれています。UIから独立したモジュールです。

## 設定管理

アプリケーションの設定は、プロジェクトのルートディレクトリにある`config.ini`ファイルで一元管理されています。APIキーや手数料率、監視対象の通貨ペアなどのパラメータは、コードを直接変更することなく、このファイルで設定・変更してください。

## 依存関係の管理

プロジェクトの全ての依存関係は、ルートディレクトリにある単一の`requirements.txt`ファイルで管理されています。

新しいライブラリを追加する際は、このファイルに追記してください。

## 開発ルール

-   **開発言語:** このプロジェクトの主要な開発言語は **Python** です。
-   **UIフレームワーク:** Web UIの開発には **Streamlit** を第一候補とします。
-   **関心事の分離:** ビジネスロジックとUIの責務分離を維持してください。
    -   普遍的なビジネスロジックや計算処理は`/core`ディレクトリに追加します。
    -   UIの表示やユーザーインタフェースに関連するコードは、`index.py`または`/pages`内の適切なファイルに配置します。
-   **コードの再利用:** UI層（`index.py`, `/pages`）から`core`の機能を呼び出すことは推奨されますが、その逆（`core`からUI層の機能を呼び出すこと）は避けてください。
